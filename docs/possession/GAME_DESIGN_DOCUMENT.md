## 1. 遊戲概述

**遊戲名稱：**（待定）

**類型：** 文字冒險 RPG，接入 AI 大語言模型即時生成動態內容

**核心玩法：**
- 玩家在由大模型驅動的世界中探索、對話戰鬥、解謎
- 所有系統模組化、配置驅動，策劃可通過修改 JSON 調整平衡
- 每次遊玩體驗因 RNG、動態生成、角色狀態變化而不同

**技術棧（預設）：**
- 後端：Python + FastAPI
- 前端：Streamlit（已建立）
- 大模型：OpenAI API / Claude API / 本地模型（Ollama）
- 數據庫：PostgreSQL（用於玩家備份快照）

**適合接駁的模型：**
- GPT-4o / Claude 3.5 Sonnet（高質量生成）
- GPT-3.5 Turbo / Ollama 7B（成本敏感場景）

## 2. 核心系統架構

### 2.1 設計模式

```
┌─────────────────────────────────────────┐
│              遊戲狀態機 (GameState)       │
│  管理玩家全局狀態、切換子系統             │
└──────────┬──────────────────────────────┘
           │
  ┌────────┼────────┬──────────┬──────────┐
  ▼        ▼        ▼          ▼          ▼
[戰鬥]  [對話]  [探索]    [副本]    [系統]
戰鬥狀態  對話狀態  移動/互動  異界狀態  存檔/載入
```

- **事件驅動架構**：各系統通過事件總線通信，如「物品掉落」「隊友叛變」「狀態變化」等事件統一分發

- **LLM 封裝層**：統一的 `LLMClient` 類，處理重試、緩存、流式輸出

## 3. 角色創建系統

### 3.1 流程

```
┌──────────────────┐
│  選擇創角方式     │
└────┬─────────────┘
     │
  ┌──┴───────────────┐
  ▼                   ▼
[隨機生成]         [手動選擇]
  │                   │
  ▼                   ▼
大模型生成背景      從預設池選擇
+ 優缺點            背景 → 優缺點 → 屬性
+ 屬性推薦            │
  │                   │
  └────────┬──────────┘
           ▼
      確認角色卡
```

### 3.2 隨機背景生成 Prompt

```
你是一個末日/奇幻 RPG 的角色背景生成器。請生成一個角色背景，
包含以下 JSON 字段：
- background_name: 背景名稱
- description: 一句背景描述
- traits_positive: [ {name, effect}, ... ]（2 個正面特質）
- traits_negative: [ {name, effect}, ... ]（2 個負面特質）
- stats: {力量, 感知, 體質, 魅力, 智力, 敏捷}（總和 20）
```

### 3.3 屬性系統（SPECIAL-like）

| 屬性 | 影響範圍 |
|------|---------|
| 力量 (STR) | 近戰傷害、負重、破門 |
| 感知 (PER) | 發現隱藏物品、陷阱、遠程命中 |
| 體質 (END) | 生命值、毒素抗性、疾病抵抗 |
| 魅力 (CHA) | 說服NPC、交易價格、招募隊友 |
| 智力 (INT) | 技能學習、破解謎題、修理成功率 |
| 敏捷 (AGL) | 閃避、開鎖、潛行、先手順序 |

**初始分配：** 總計 20 點，可手動分配或隨機擲骰

### 3.4 特質池（部分示例）

**正面特質：**
| 名稱 | 效果 |
|------|------|
| 鐵胃 | 可食用腐爛食物不生病 |
| 社交蝴蝶 | NPC 初次對話好感 +20 |
| 霉菌抗體 | 接觸霉菌時不變菌人，獲臨時增益 |
| 次元敏感 | 次元穿梭冷卻減半 |
| 急救精通 | 治療道具效果 +30% |

**負面特質：**
| 名稱 | 效果 |
|------|------|
| 路痴 | 地圖移動時 20% 概率進入隨機戰鬥 |
| 過敏體質 | 被昆蟲攻擊時額外受傷 |
| 心軟 | 面對受傷敵人時，攻擊力 -20% |
| 體臭 | 隊友忠誠度初始 -10 |
| 物資依賴 | 無醫療包時，意志檢定 -2 |

## 4. 戰鬥前對話系統

### 4.1 觸發流程

```
遭遇敵人
  │
  ▼
生成敵人開場白（大模型）
  │
  ▼
顯示：敵人描述 + 對話選項
  │
  ├──攻擊──→ 先手攻擊一次，進入戰鬥
  ├──說服──→ 魅力檢定 → 成功則避戰
  ├──威脅──→ 力量/名聲檢定 → 成功則敵人逃跑
  ├──觀察──→ 感知檢定 → 獲得敵人弱點（戰鬥傷害+20%）
  └──逃跑──→ 敏捷檢定 → 成功則脫離
```

### 4.2 說服/威脅成功率公式

```
成功率 = 基礎值(50%) + (屬性修正) + (特質加成) + (背景加成)
  魅力修正：每高於 5 一點 +5%
  特質加成：如「社交蝴蝶」+15%
  背景加成：如「廢土醫生」面對受傷敵人 +10%
  封頂：95%  最低：10%
```

### 4.3 大模型生成格式

```json
{
  "enemy_name": "掠奪者首領",
  "appearance": "穿著破爛皮甲的男人，臉上有燒傷疤痕",
  "opening_line": "「把身上的藥交出來，不然別怪我下手狠。」",
  "persuasion_success_response": "「……你是醫生？好吧，我弟弟上次也是被你們這類人救的。滾。」",
  "persuasion_fail_response": "「廢話真多！」",
  "threat_success_response": "「別…別殺我！我這就走！」",
  "weakness": "他右腿有舊傷，移動速度較慢"
}
```

## 5. 動態 NPC 生成

### 5.1 觸發場景

- 玩家進入廣場/酒館等公共區域
- 選擇「與隨機 NPC 交談」
- 大模型實時生成一個新 NPC

### 5.2 生成 Prompt

```
你是一個奇幻 RPG 的廣場 NPC 生成器。請生成一個願意與玩家
交談的 NPC，包含：
- name: 名字（中文，2~4 字）
- appearance: 外貌描述（一句話）
- personality: 性格關鍵詞（1~2 個）
- quest: 發布的任務（可選，如無則填 null）
- reward: 任務獎勵描述
- flavor_text: 見面時的開場白

輸出 JSON 格式。
```

### 5.3 NPC 生成後可觸發

| 結果類型 | 說明 |
|---------|------|
| 發布任務 | NPC 給予支線任務，完成後獲得獎勵 |
| 觸發戰鬥 | NPC 挑釁或顯露敵意 |
| 獲得信息 | NPC 透露地圖秘密/隱藏商店位置 |
| 隨機獎勵 | 純對話，NPC 給予小禮物 |
| 招募前置 | NPC 需要完成任務後才願意加入 |

## 6. 隊友系統

### 6.1 數據結構

```json
{
  "companion_id": "c_001",
  "name": "流浪樂師·阿菲婭",
  "appearance": "揹著破舊魯特琴，左眼有一道傷疤",
  "personality": "表面玩世不恭，實則渴望被信任",
  "join_condition": "玩家幫她贖回被當掉的琴譜",
  "special_skill": "演奏激勵（臨時提升隊友命中）",
  "hidden_trait": {
    "type": "間諜",
    "value": 0.7,
    "clue_line": "「如果…形勢不對，你可別怪我第一個跑。」",
    "is_betrayer": false
  },
  "loyalty_score": 50
}
```

### 6.2 隱藏特質類型

| 類型 | 基礎背叛概率 | 觸發條件 |
|------|------------|---------|
| 間諜 | 80% | 忠誠度 < 30 → 必然叛變 |
| 懦夫 | 60% | 玩家在最終戰死亡過一次 → 逃跑 |
| 忠犬 | 5% | 除非玩家親手殺死其摯友 |
| 野心家 | 70% | 玩家放棄王座 → 取而代之 |
| 詛咒者 | 30~90% | 受詛咒反噬，可能無差別攻擊 |

### 6.3 結局判定邏輯

```
最終背叛概率 =
  基礎概率(類型) + (50 - 忠誠度) / 100

if random() < 最終概率:
    觸發叛變結局
else:
    觸發忠誠結局
```

## 10. 次元穿梭系統

### 10.1 觸發方式

| 方式 | 說明 |
|------|------|
| 主動技能 | 消耗「次元能量」/「精神力」，需解鎖 |
| 被動觸發 | 特定地點（古井、星盤陣）或持有「破碎懷表」時概率觸發 |
| 隨機裂縫 | RNG 事件表（5% 概率，無需消耗能量） |

### 10.2 場景生成 Prompt

```
你是一個奇幻 RPG 的「次元場景生成器」，生成一個小型異界場景：
- scene_name: 3~6 字名稱（如「萬鏡回廊」）
- atmosphere: 氛圍關鍵詞
- description: 50 字內場景描述
- interactables: [ {名稱, 描述, 互動類型} × 3 ]
- secret: { type, content }
- time_cost_per_action: 默認 1

輸出 JSON。
```

### 10.3 穿梭流程

```
使用能力
  │
  ▼
大模型生成場景 JSON
  │
  ▼
顯示場景 + 可互動對象 + 倒計時（5 回合）
  │
  ├──與對象互動──→ 消耗 1 回合，觸發戰鬥/對話/拾取
  ├──尋找隱藏秘密──→ 消耗 1 回合，可能遇到特殊隊友
  └──主動離開──→ 消耗 1 回合，返回原世界
      │
  倒計時歸零 ──→ 強制傳送
```

### 10.4 異界獎勵

- **特殊道具**：僅在異界內有效，返回後失效（或轉化為永久道具）
- **特殊隊友**：可帶回主線，帶有隱藏特質
- **次元碎片**：收集 5 塊解鎖「終極次元穿梭」，進入隱藏結局

## 11. 奪舍系統

### 11.1 設計原則

- 非實時 PvP：不影響原玩家遊玩
- 定期備份：每 12 小時為活躍玩家生成快照
- 新玩家可選「奪舍」一個備份角色，觀看其殘存記憶並繼續劇情

### 11.2 數據庫表

```sql
CREATE TABLE player_snapshots (
    id UUID PRIMARY KEY,
    original_player_id UUID,
    snapshot_time TIMESTAMP,
    character_json JSONB,
    recent_events TEXT[],
    last_words TEXT,           -- AI 生成的一句遺言
    is_public BOOLEAN DEFAULT true
);
```

### 11.3 奪舍後「殘響系統」

新玩家可在營地冥想，回放被奪舍者的記憶片段（由大模型根據事件日誌生成）。選擇影響新玩家屬性或隱藏值。

### 11.4 與其他系統聯動

| 場景 | 效果 |
|------|------|
| 被奪舍者曾接觸霉菌 | 新玩家直接獲得霉菌抗性或直接變菌人 |
| 被奪舍者有「間諜」特質 | 新玩家奪舍後間諜線索仍可能觸發 |
| 被奪舍者進入過某異界 | 新玩家解鎖該異界「坐標碎片」 |

## 13. 修理系統 ✅ 已實現

> **代碼：** `game/repair_system.py` + `data/repair_system.json`
> **狀態：** 完整實現，40 個單元測試通過

### 13.1 耐久度機制

所有武器/防具配置最大耐久，使用/受擊消耗耐久。歸零後失效（可修理）。

```json
{
  "rusty_sword": {
    "max_durability": 50,
    "damage_per_use": 1,
    "damage_on_block": 2,
    "repair_material": "scrap_metal",
    "repair_material_amount": 2,
    "repair_skill": "smithing"
  }
}
```

### 13.2 修理方式

| 方式 | 成功率 | 消耗 | 適用場景 |
|------|--------|------|---------|
| 自己修理 | 60% + 技能修正 | 材料 + 工具 | 有材料時隨時 |
| NPC 修理 | 100% | 金幣（按損失比例） | 城鎮/營地鐵匠 |
| 霉菌修理 | 80% | HP（菌人專用） | 有機材質裝備 |
| 次元修理 | 100% | 消耗一次穿梭機會 | 異界特殊事件 |

### 13.3 修理成功率公式

```
成功率 = 基礎(60%) + (智力 - 5) × 5% + 技能點 × 5%
封頂：95%  最低：10%
```

## 17. 大模型調用策略

### 17.1 調用封裝層

```python
class LLMClient:
    def __init__(self, provider="openai", model="gpt-4o"):
        self.provider = provider
        self.model = model
        self.cache = {}

    def generate(self, prompt: str, system: str = "",
                 json_mode: bool = False,
                 max_tokens: int = 500) -> str:
        # 1. 檢查緩存
        cache_key = hash((system, prompt))
        if cache_key in self.cache:
            return self.cache[cache_key]

        # 2. 調用 API（帶重試）
        response = self._call_with_retry(...)

        # 3. 解析/緩存
        if json_mode:
            return json.loads(response)
        return response
```

| 記憶片段生成 | GPT-3.5 | 300 | ✅ |

## 18. 維護與調試建議

### 18.1 新增 RNG 事件

只需在 `random_encounters.json` 新增一條：

```json
{ "id": "treasure_chest", "weight": 8, "type": "loot", "min_gold": 10 }
```

代碼無需修改，自動生效。

### 18.2 新增修理方式

1. 在 `repair_system.json` 的 `special_repairs` 新增條目
2. 在 `RepairSystem.attempt_repair()` 加一個 `elif` 分支
3. 對應物品的 `repair_type` 設為新的修理類型

### 18.3 關閉隨機性（調試模式）

在 `openclaw.json` 或遊戲設定中加入：

```json
{
  "debug": {
    "seed": 42,
    "disable_random": true,
    "show_all_rolls": true
  }
}
```

所有隨機改為固定 seed，輸出詳細骰點日誌。

### 18.4 快速測試特定劇情

在存檔中直接設置狀態，跳過正常流程：

```json
{
  "player": {
    "is_mancelled": true,
    "companions": [{ "id": "c_001", "loyalty_score": 20 }]
  }
}
```
